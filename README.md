# AI Shared Config

Shared Claude Code configuration, agents, hooks, guidelines, and GitHub Actions workflows for all Laravel projects.

## How It Works

This repo is added as a **Git submodule** at `.ai-shared/` in each Laravel project. A setup script symlinks the contents into the locations Claude Code and GitHub expect (`.claude/`, `.github/`, `.ai/`).

```
your-project/
├── .ai-shared/          ← this repo (submodule)
├── .claude/             ← symlinked from .ai-shared
├── .github/             ← symlinked from .ai-shared
├── .ai/guidelines/      ← symlinked from .ai-shared
├── AGENTS.md            ← PROJECT-SPECIFIC (not from this repo)
└── .ai-local/           ← project-specific overrides (gitignored in this repo)
```

## First-Time Setup (New Project)

```bash
# Add the submodule
git submodule add git@github.com:YOUR_USERNAME/ai-shared-config.git .ai-shared

# Run the setup script
.ai-shared/scripts/setup.sh

# Create your project-specific AGENTS.md
cp .ai-shared/templates/AGENTS.md.example AGENTS.md
# Edit AGENTS.md with your project details

# Commit
git add .ai-shared .claude .github .ai AGENTS.md .gitignore
git commit -m "chore: add shared AI config"
```

## Cloning a Project That Already Has This

```bash
git clone --recurse-submodules git@github.com:YOUR_USERNAME/your-project.git

# Or if you already cloned without --recurse-submodules:
git submodule update --init --recursive
.ai-shared/scripts/setup.sh
```

## Updating Across All Projects

Edit this repo, push, then in each project:

```bash
.ai-shared/scripts/update.sh
```

Or update all projects at once with the bulk script:

```bash
# From a parent directory containing all your projects
.ai-shared/scripts/update-all.sh ~/Code
```

## Structure

```
ai-shared-config/
├── claude/
│   ├── agents/              # Sub-agents (reviewer, simplifier, test-writer)
│   ├── commands/             # Slash commands (/review, /simplify, /test)
│   └── hooks.json            # Pre-commit quality gates
├── github/
│   └── workflows/
│       ├── claude-review.yml # AI code review on PRs
│       └── tests.yml         # Standard Laravel test pipeline
├── guidelines/
│   ├── php-laravel.md        # PHP & Laravel coding standards
│   └── pest-testing.md       # Pest testing conventions
├── templates/
│   ├── AGENTS.md.example     # Starting point for project AGENTS.md
│   └── AppServiceProvider.example.php  # Strict defaults
├── scripts/
│   ├── setup.sh              # First-time symlink setup
│   ├── update.sh             # Pull latest + re-link
│   └── update-all.sh         # Bulk update across projects
└── README.md
```

## Project-Specific Overrides

If a project needs extra agents or guidelines beyond the shared ones, put them in `.ai-local/`:

```
your-project/
├── .ai-local/
│   ├── agents/
│   │   └── filament-expert.md   # Project-specific agent
│   └── guidelines/
│       └── filament.md          # Project-specific guidelines
```

The setup script merges `.ai-local/` into the symlinked directories so both shared and local configs are available.

## What Stays Project-Specific

These files are NEVER shared and always live in each project:

- `AGENTS.md` — project architecture, packages, conventions
- `.mcp.json` — generated by `php artisan boost:install`
- `.ai-local/` — project-specific overrides
